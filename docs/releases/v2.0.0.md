# Release Notes — v2.0.0

**Release Date:** 2026-04-XX (TBD)  
**Release Candidate:** v2.0.0-rc1  
**Status:** Draft — Subject to revision before final release

---

## Overview

Phoenix Trade Bot v2.0.0 is a **major release** that removes deprecated components, introduces a new pipeline-based execution engine, and hardens the platform for production go-live. This release includes breaking changes that require migration steps before upgrading.

---

## Highlights

### Phase A: Pipeline Engine + Broker Integration
- **New pipeline-based execution architecture**: Signal ingestion → enrichment → inference → risk check → broker execution, all orchestrated through Redis Streams.
- **Robinhood MCP integration**: Live trading via Robinhood MCP server with session persistence and zero-latency credential restore.
- **IBKR Gateway support**: Interactive Brokers execution via IB Gateway with daily re-auth flow.
- **Removed Alpaca support**: Alpaca broker adapter deprecated and removed (breaking change).
- **Engine type required**: All new agents must specify `engine_type` field (`pipeline` or `legacy`) in creation requests (breaking change).

### Phase B: Observability and Monitoring
- **Grafana dashboards**: Pre-built agent wake-flow dashboard at `infra/observability/grafana/agent-wake-flow.json`.
- **Prometheus metrics**: Agent lifecycle metrics, broker adapter circuit breaker states, Redis stream lag.
- **Structured logging**: JSON-formatted logs with request IDs and trace context.
- **Heartbeat monitoring**: Agent heartbeat endpoint with `last_activity_at` tracking.

### Phase C: Backtesting Database Persistence
- **Backtest artifact storage**: All backtesting runs now persist metrics, model files, and evaluation reports to `agent_backtest_artifacts` table.
- **Historical comparison**: UI surfaces p50/p95 latency trends across multiple backtest runs.
- **Reproducibility**: Every backtest tagged with git SHA, Python version, and dependency snapshot.

### Phase D: Dashboard UX Improvements
- **Agent detail tabs**: Redesigned 10-tab layout with Trades, Backtesting, Skills, Logs, and Config tabs.
- **Decision trail visibility**: Eye icon expands full `decision_trail` JSON for every trade.
- **Real-time updates**: SSE streaming for agent logs and trade feed.

### Phase F: OpenClaw Removal (Breaking Change)
- **OpenClaw Bridge Service removed**: All `openclaw/bridge/` code deleted; no remote agent execution path.
- **Makefile targets removed**: `run-bridge` and `test-bridge` targets no longer available.
- **Environment variable cleanup**: `BRIDGE_TOKEN` no longer used; remove from `.env`.

## Breaking Changes

### 1. OpenClaw API Removed
**Impact:** All OpenClaw Bridge Service endpoints (`/api/v2/bridge/*`) return 404. Remote agent execution via OpenClaw is no longer supported.

**Migration:**
- Remove `BRIDGE_TOKEN` from `.env`.
- Remove any references to `openclaw/bridge/` in deployment scripts.
- Agents must run locally via Agent Gateway (`/api/v2/agents`).

### 2. `engine_type` Required Field
**Impact:** `POST /api/v2/agents` now requires `engine_type` field (`"pipeline"` or `"legacy"`). Requests without this field will return 400 Bad Request.

**Migration:**
```python
# Before (v1.x)
{
  "name": "My Agent",
  "type": "trading",
  "connector_id": "..."
}

# After (v2.0)
{
  "name": "My Agent",
  "type": "trading",
  "connector_id": "...",
  "engine_type": "pipeline"  # NEW: required field
}
```

### 3. Alpaca Support Removed
**Impact:** `AlpacaBrokerAdapter` class deleted. Any agents configured with Alpaca connectors will fail to execute trades.

**Migration:**
- Switch to Robinhood MCP or IBKR Gateway.
- Update agent connector configurations in the database.
- Re-run backtesting with new broker adapter.

### 4. Makefile Targets Removed
**Impact:** `make run-bridge` and `make test-bridge` no longer exist.

**Migration:**
- Remove any CI/CD or deployment scripts that reference these targets.
- Use `make run-api` and `make test-api` instead.

## Migration Steps

### Prerequisites
- Python 3.11+ (Python 3.9 local test environments are no longer supported)
- PostgreSQL 14+
- Redis 7+

### 1. Backup Database
```bash
pg_dump -U postgres phoenix_trade_bot > backup_v1.15.3.sql
```

### 2. Update Environment Variables
```bash
# Remove deprecated vars
unset BRIDGE_TOKEN

# Add new vars (if using pipeline engine)
export ROBINHOOD_MCP_URL="http://localhost:8012"
export IB_GATEWAY_HOST="localhost:4001"
```

### 3. Run Database Migrations
```bash
# Verify current head
make db-alembic-heads
# Expected output: 038_decision_trail (head)

# Apply new migrations (046-048)
make db-upgrade
```

### 4. Update Agent Configurations
All existing agents must have `engine_type` field added:
```sql
-- Add engine_type to existing agents (default to legacy for backward compatibility)
UPDATE agents SET engine_type = 'legacy' WHERE engine_type IS NULL;
```

### 5. Rebuild and Restart Services
```bash
# Pull latest code
git fetch origin
git checkout v2.0.0

# Rebuild Docker images
docker compose build

# Restart services
docker compose up -d
```

### 6. Verify Deployment
```bash
# Check API health
curl https://your-host/health

# Verify agents list loads
curl -H "Authorization: Bearer $JWT_TOKEN" https://your-host/api/v2/agents

# Check database migration status
make db-current
```

## Known Issues

### 1. Migration 049 Pending 7-Day Verification
- Migration `049_pipeline_engine_phase_a` includes a data migration that requires 7-day soak period before promotion to production.
- **Action:** Monitor logs for errors related to `engine_type` NULL constraint violations.

### 2. IBKR Daily Re-Auth Required
- IB Gateway requires daily re-authentication due to token expiration.
- **Workaround:** Schedule cron job to restart IB Gateway at 5 AM ET daily, or manually re-auth via TWS.

### 3. Python 3.9 Local Test Environments Not Supported
- CI/CD and local development require Python 3.11+.
- **Action:** Upgrade local Python environments to 3.11 or use Docker-based test runner.

## Rollback Plan

If critical issues are discovered after deployment, follow these rollback steps:

### Per-Phase Rollback

#### Phase A (Pipeline Engine)
```bash
# 1. Downgrade database migration
alembic downgrade 045  # Back to pre-pipeline state

# 2. Revert code to v1.15.3
git checkout v1.15.3

# 3. Rebuild and restart
docker compose up -d --build
```

#### Phase B (Observability)
```bash
# No database changes; simply revert code
git checkout v1.15.3
docker compose restart phoenix-api
```

#### Phase C (Backtesting DB)
```bash
# 1. Downgrade migration 047
alembic downgrade 046

# 2. Revert code
git checkout v1.15.3
docker compose up -d --build
```

#### Phase F (OpenClaw Removal)
```bash
# 1. Restore openclaw/bridge/ from v1.15.3
git checkout v1.15.3 -- openclaw/bridge/

# 2. Restore Makefile targets
git checkout v1.15.3 -- Makefile

# 3. Re-add BRIDGE_TOKEN to .env
echo "BRIDGE_TOKEN=your_token_here" >> .env

# 4. Rebuild
docker compose up -d --build
```

### Full Rollback to v1.15.3
```bash
# 1. Restore database backup
psql -U postgres phoenix_trade_bot < backup_v1.15.3.sql

# 2. Revert code
git checkout v1.15.3

# 3. Rebuild and restart
docker compose up -d --build

# 4. Verify health
curl https://your-host/health
```

- Pipeline-based execution engine with Redis Streams orchestration
- Robinhood MCP broker adapter with session persistence
- IBKR Gateway broker adapter
- `engine_type` field on `agents` table (migration 046)
- `agent_backtest_artifacts` table for backtest persistence (migration 047)
- Grafana dashboard for agent wake-flow monitoring
- Prometheus metrics for circuit breaker states and stream lag

- Staged rollout plan (`docs/operations/go-live-rollout-plan.md`)
- Observability checklist (`docs/operations/go-live-observability.md`)

## Changed

- `POST /api/v2/agents` now requires `engine_type` field (breaking change)
- Agent lifecycle state machine extended with `backtesting_approved` state
- Dashboard agent detail page redesigned with 10 tabs

- **OpenClaw Bridge Service** (`openclaw/bridge/` directory deleted)
- **Alpaca broker adapter** (`shared/broker_adapters/alpaca.py` removed)
- **Makefile targets**: `run-bridge`, `test-bridge`
