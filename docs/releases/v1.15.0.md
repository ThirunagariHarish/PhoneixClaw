## Highlights

Phase 15 delivers a fully autonomous **Prediction Markets** feature on top of the
existing Phoenix Trade Bot platform. The system can discover prediction-market
contracts, score them with a four-stage AI pipeline, surface the top bets in the
dashboard, and let users converse with an AI about any bet — all without ever
placing real money (paper mode is enforced at the adapter layer).

## What's New

### AI Scorer Chain
The heart of Phase 15 is a four-stage scoring pipeline applied to every candidate
prediction-market contract:

1. **ReferenceClassScorer** — locates historically similar resolved markets via
   embedding cosine similarity; derives a prior probability (base rate).
2. **CoTSampler (N=5)** — fires five parallel LLM chain-of-thought prompts and
   combines them for self-consistency; produces a probability distribution.
3. **DebateScorer** — a Bull agent argues YES, a Bear agent argues NO, and a Judge
   LLM adjudicates; outputs structured `bull_argument`, `bear_argument`,
   `debate_summary`.
4. **LLMScorer + TopBetScorer** — blends all upstream signals into a final
   `confidence_score` (0–100) and `edge_bps`; gates bets below threshold.

### TopBetsAgent (24/7 Autonomous)
Runs inside the `pm-top-bets-agent` Docker service (opt-in, `--profile agents`).
- 60-second scoring cycle over the full venue market catalogue.
- Redis heartbeat key `pm:top_bets_agent:heartbeat` with 90-second TTL.
- Registered with the Phoenix orchestrator for unified health monitoring.

### AutoResearchAgent (Daily)
Nonce-gated to run once per calendar day. Identifies market categories with
low historical coverage, generates research queries, and stores findings in
`pm_strategy_research_log` for human review before any config changes are applied.

### Venue Adapters (Paper Mode Only)
Both `RobinhoodPredictionsVenue` and `PolymarketVenue` enforce paper mode at the
adapter layer. `place_order(paper=False)` raises `ValueError` unconditionally —
there is no code path to live trading in Phase 15.

### Dashboard
- **"Prediction Markets"** tab (renamed from "Polymarket").
- **VenueSelector** pills — switch between Robinhood Predictions and Polymarket.
- **TopBetsPanel** — ranked bet cards with confidence bars and expandable AI
  reasoning (bull/bear/debate summary).
- **Chat tab (8th)** — SSE streaming chat grounded on current top bets and market
  context.
- **Logs tab (9th)** — real-time agent activity feed, health badge, manual
  cycle-trigger button.
- `JurisdictionBanner` updated to blue/info style.

### Infrastructure
- `pm-top-bets-agent` Docker service under `profiles: [agents]` — plain
  `docker compose up` is unaffected.
- `.env.example` extended with `PM_TOP_BETS_ENABLED`, `PM_TOP_BETS_VENUE`,
  `PM_TOP_BETS_CYCLE_INTERVAL_S`, `PM_RESEARCH_ENABLED`.
- `_ensure_prod_schema()` safety net on agent startup — prevents schema-drift
  crashes in environments where migrations ran partially.

docker compose --profile agents up -d pm-top-bets-agent

## Rollback Plan

### Quick rollback (< 5 minutes)

```bash
# 1. Stop the new agent service (if started)
docker compose --profile agents stop pm-top-bets-agent

# 2. Revert to the previous release image / code
git checkout v0.5.0
docker compose up -d --build
```

### Full rollback (including DB)

```bash
# 1. Stop services
docker compose --profile agents stop pm-top-bets-agent
docker compose stop phoenix-api

# 2. Downgrade DB (drops all 7 pm_phase15 tables — no existing data affected)
alembic downgrade 032

# 3. Revert code and rebuild
git checkout v0.5.0
docker compose up -d --build phoenix-api
```

> **Data note:** The 7 new tables introduced in migration 033 contain only data
> generated after upgrade. Downgrading drops those tables entirely. No pre-existing
> data is modified at any point.
