## Deployment Steps

> **Order matters.** Run the migration before restarting the API service.

```bash
# 1. Pull the new image / code
git pull origin main   # or deploy via your CD pipeline

# 2. Run the DB migration (idempotent â€” safe to re-run)
alembic upgrade head

# 3. Restart the API service
#    Docker Compose:
docker compose restart phoenix-api

#    Coolify / standalone:
systemctl restart phoenix-api   # or equivalent

# 4. Verify analyst endpoints are live
curl -s http://localhost:8000/api/v2/signals | jq .
```

### Environment variables â€” no new ones required

All analyst-agent configuration is driven by `PersonaConfig` at agent-creation
time. No new `.env` entries are needed for Phase 1.

## Rollback Procedure

If a critical issue is found after deployment:

```bash

# 2. Check out the previous release tag
git checkout v0.4.0

# 3. Restart the API service
docker compose restart phoenix-api
```
