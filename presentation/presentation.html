The Problem

Trading Bots Are Broken

Hard-Coded Rules

Traditional bots rely on rigid if/else logic that can't adapt to changing markets.

Raw LLM Calls

Newer bots let LLMs make discretionary trades — unpredictable, untested, and dangerous.

No Backtesting Gate

Strategies go live without proof. No win rate validation, no Sharpe ratio, no drawdown limits.

Single Point of Failure

Monolithic architecture — one crash takes down everything. No risk isolation.

The Solution

LLMs as Strategy Engineers, Not Discretionary Traders

ProjectPhoenix is an autonomous multi-agent trading platform where AI agents must prove themselves through backtesting before touching real capital.

Core Principle: Every agent must pass rigorous backtest gates — win rate, Sharpe ratio, profit factor, max drawdown — before promotion to live trading.

Key Idea

Design Principles

🧠 Intelligence in Agents

Dashboard is control & display only. All reasoning, strategy, and trading logic runs inside OpenClaw AI agents.

🔒 Queue-Based Execution

Agents never call broker APIs directly. Trade intents go through a validated queue with risk checks at every layer.

🤝 Paired Agents

Every trading agent is paired with a monitoring agent responsible for exits, stop-losses, and position management.

🔄 Self-Improving System

Dev Agent with reinforcement learning monitors performance, fixes issues, tunes parameters, and evolves strategies.

Key Idea

A Trading Firm of AI Agents

Think of Phoenix not as a bot, but as an entire trading firm staffed by specialized AI agents:

  • Research Analysts — ingest signals from Discord, Twitter, Unusual Whales
  • Strategy Engineers — design and backtest trading strategies
  • Traders — execute approved strategies in live markets
  • Risk Managers — monitor positions, enforce limits, trigger kill switch
  • Dev Agent — CTO that improves the firm's code and models

End-to-End Flow

How a Trade Happens

Step-by-step journey from signal to execution

1
Signal Arrives — A Discord message, tweet, or Unusual Whales alert hits the Connector Manager.
2
Ingestion & Normalization — Connector parses, extracts tickers (NLP), sentiment, and publishes to Event Bus.
3
Agent Evaluation — Strategy Agent receives signal, applies backtested strategy with LLM reasoning.
4
Trade Intent Created — If signal passes evaluation, agent posts a trade intent to the execution queue.
5
3-Layer Risk Check — Agent risk → Execution risk → Global risk. Any layer can reject.
End-to-End Flow

How a Trade Happens (cont.)

6
Broker Execution — Execution Service places order via Alpaca/IBKR API. Idempotent with unique trade IDs.
7
Position Created — Order filled, position recorded in DB. Monitoring Agent automatically paired.
8
Real-Time Updates — WebSocket pushes position updates, P&L, and status to Dashboard instantly.
9
Position Monitoring — Monitoring Agent tracks price, checks stop-loss, take-profit, and exit conditions.
10
Dev Agent Learns — RL loop analyzes trade outcome, adjusts parameters, re-backtests improved strategies.
End-to-End Flow
